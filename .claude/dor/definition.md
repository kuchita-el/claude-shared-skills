# Definition of Ready (DoR) — プロジェクト固有定義

デフォルトDoRの共通項目に加えて、このプロジェクトで追加するチェック項目です。

## 追加チェック項目

### Medium

**対象**: 機能追加、既存機能の改善、リファクタリングなど

デフォルトのチェック項目に加えて:

| 項目                               | 説明                                                                                       |
| ---------------------------------- | ------------------------------------------------------------------------------------------ |
| 背景・目的が明確                   | なぜこの変更が必要なのかが説明されている                                                   |
| 要件が具体的                       | 実装すべき内容が具体的に記載されている。「〜など」「適切に」のような曖昧な表現や、複数解釈可能な箇所がない |
| 受け入れ条件（AC）が定義されている | テスト可能な形式で完了条件が列挙されている                                                 |
| 見積もり可能                       | 技術方針や実装方式が十分に明確で、工数の見積もりができる                                   |
| 不明点がクリアになっている         | 未決定の設計判断（技術選定、方式の選択等）が残っていない                                   |
| スコープが適切                     | 1PR（1-2日）で完了できる規模である。超える場合は分割案がある                               |
| 前提条件が明確                     | 先に解決すべき問題やブロッカーがない                                                       |

### Large

**対象**: アーキテクチャ変更、大規模機能追加、設計変更など

Mediumのチェック項目に加えて:

| 項目                         | 説明                                           |
| ---------------------------- | ---------------------------------------------- |
| トレードオフが検討されている | 設計の選択肢と選定理由が記載されている         |
| 技術的依存関係が明確         | 他モジュール・サービスへの影響が特定されている |
| UI/UX の決定がされている     | 該当する場合、画面設計・フローが確定している   |
| 分割案（子Issue）がある      | Medium/Small サイズの子Issueに分割されている   |
