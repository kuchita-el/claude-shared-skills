# ADR-20260627-2: plan-issue の plan 本文をメイン context に往復させない生成〜修正ループ

## Context

`plan-issue` は plan サブエージェントが生成した **plan 本文全文**をメインへ返し、メインがそれを Write で保存（SKILL.md:94）、自動レビュー FAIL 時は Edit で修正してレビューを再ループする（SKILL.md:119、最大2周）。このため plan 本文がメイン context に着弾し、保存・修正の双方で参照されるため滞留し続ける。plan の規模に上限定義がなく、最大2周のループで複数回登場しうるため、スキル群の中で最大の単一 context 消費源となっている。一方、レビュアー呼び出し（SKILL.md:109）は既に **plan ファイルのパスのみ**を渡し、`plan-reviewer` が自前で Read する良い型であり、本決定はこの型を生成・保存・修正経路へ拡張する。

先行する ADR-20260627（コンテキスト予算規約、`context-budget.md`）が「サブエージェント返却のサマリ化」「パス渡し原則」「生ツール出力のファイル化」「滞留カテゴリ規則（メイン滞留は SKILL.md 本体＋構造化サマリのみ）」を単一出典として確定し、その Context で「plan-issue の plan 本文往復」を未解決の横断パターンとして明示した（具体設計は本 ADR へ委譲）。本決定はその傘規約を plan-issue に適用する最初の具体ケースである。

設計上の制約として、(1) `plan` エージェントは現状 Read/Grep/Glob のみの読み取り専用で、「保存はスキル側が行い、エージェントは本文をテキストで返すことに徹する」と意図的に規定されている（plan.md:21）、(2) ユーザースコープ CLAUDE.md がサブエージェントの「実行ログを最終出力に含める」記録義務を課す、(3) 連番衝突回避命名（SKILL.md:96）が人間却下後リカバリの再生成時に旧 plan ファイルへのパス参照を温存する、という既存の振る舞いを壊さないことが求められた。

## Decision

plan 本文をメイン context に往復・滞留させないため、生成〜保存〜自動レビュー〜修正ループの責務を以下のとおり再配置する。

1. **A. 生成エージェントが plan ファイルを直接 Write する（plan 本文をメインへ返さない）**
   `plan` エージェントに Write/Edit 権限を付与し、エージェント自身がプランファイルへ書き込む。メインへの返却は **パス＋構造化サマリ**に限定し、plan 本文全文は返さない。これにより plan.md:21 の読み取り専用方針を意図的に改め、エージェントの責務を「plan 成果物の生成」から「plan 成果物の生成および レビュー駆動の改訂を所有するライフサイクル」へ拡張する。独立レビュアーによる検証（自己評価バイアスの分離）は維持される。分割モード（`split-plan-procedure.md`）の各詳細プラン（初回 step2・N回目 step3-4 で plan サブエージェントが生成し、現行はメインが Write 保存）も同様に、生成エージェントが直接 Write し本文をメインへ返さない。一方、分割計画マスター（`issue-{番号}-plan.md`、step1）は AC↔PR マトリクスの小型の俯瞰計画で、現状 step1 は plan サブエージェントを明示せずメイン側生成と読める。最大の単一 context 消費源である詳細プラン本文と比べ滞留影響が小さいため、本決定では対象を詳細プランに限定し、マスター生成主体は変更しない。

2. **B. 連番衝突回避命名はメインが Glob 単独で行う**
   メインが `docs/plans/` を Glob で列挙して空き連番名を決定論的に解決し、解決済みパスをエージェントへ渡してそこへ Write させる。旧 plan ファイルを上書きしない不変条件はメイン側のコードで担保し、LLM へ委譲しない。現行の「Glob/Read で同名確認」の Read は廃する（存在確認は Glob で足り、旧 plan 本文の Read 自体が context-budget 違反となるため）。命名は生成後から生成前へ移るが、並行書き込みが無いため機能的に等価である。なお分割モード（`split-plan-procedure.md`）は step6 とは独立した命名規則を持つ。エージェントが Write する詳細プラン（`issue-{番号}-1.md`／`-k.md`、衝突時は同じく連番サフィックス）には本 B を適用し、メインが Glob で空きパスを解決してエージェントへ渡す。分割計画マスター（`issue-{番号}-plan.md`）は初回/N回目の判別子となる固定名でメインが直接生成・Write するため、B の対象外とする。

3. **C. 自動レビュー FAIL 時の修正は新インスタンスの plan エージェントへ委譲する（C-再起動）**
   FAIL 時、メインが Edit する代わりに、新しい plan エージェントインスタンスに **plan ファイルのパス＋レビュアーの指摘テーブル**を渡し、エージェントが自前で Read → 対象箇所を Edit → 変更サマリを返す。plan 本文はメインに載らない。最大2周・収束しない場合の step8 質問への接続は不変とする。

   代替案として、生成インスタンスを生かしたまま指摘を追送する「C-継続」を検討したが却下した。本システムは plan を自己完結したファイルへ外部化し、文脈を持たない新鮮なアクター（生成時セルフレビュー・独立レビュアー・人間レビュアー・人間却下後リカバリの再生成）がそれを読む型で一貫している。C-継続はこの型の中にファイルに無い私的記憶へ依存するアクターを持ち込み、アーキの木目に逆行する。また C-継続が prevent するのは FAIL 経路のサブエージェント側再 Read のみで、利得は限定的である一方、冪等性・再開性・SKILL.md の直線性・フォールバック整合・人間却下リカバリとの単一メンタルモデルはいずれも C-再起動が優位である。

4. **D. 実行ログはファイルに載せず返却にのみ含める**
   エージェントはファイルに plan 本文のみを書き、実行ログを書かない。実行ログはユーザースコープ CLAUDE ルールに従い**返却**に含める。メインは summary＋パスのみ保持し、実行ログは当該ターンで破棄する（context-budget 原則4 の net 滞留ゼロ＝一時経由は許容）。これにより「実行ログ除去」は「メインが除去する」から「エージェントが最初からファイルに載せない」へ機構が変わり、メインの除去ステップが不要になる。

## Consequences

**既存4機能の維持**（本設計での担い手）:

| 機能 | 維持方式 |
| --- | --- |
| 連番衝突回避命名（SKILL.md:96） | B。メインが Glob 単独で空き連番名を解決し解決済みパスを渡す。不変条件はメインで決定論的に担保 |
| 実行ログ除去（SKILL.md:94） | D。エージェントがファイルに本文のみ書きログを載せない。ログは返却に含めメインは破棄 |
| FAIL 時 Edit 修正（SKILL.md:119） | C-再起動。新インスタンスがパス＋指摘で自前 Read→Edit→サマリ返却。最大2周・収束不可時の質問は不変 |
| 人間却下後リカバリ（SKILL.md:142-） | 不変かつより一貫。再生成＝スキル再起動＝新インスタンスが連番ファイルを書く（A で直接 Write）。C-再起動と同一メンタルモデル。B が旧 plan 温存を継続担保 |

**得られた利益**:

- plan 本文がメイン context に一切載らず、ADR-20260627 の滞留カテゴリ規則に適合する。起動側モデルの圧縮挙動への依存が縮減する。
- 全アクターが同一の自己完結ファイルを読む一貫性が保たれ、C-再起動は「新鮮な編集者が直せる程度に plan が self-contained か」を逆に圧力テストする。
- 自動レビューの独立性（自己評価バイアスの分離）が維持される。

**受容したトレードオフ**:

- `plan` エージェントへの Write/Edit 権限付与は、plan.md:21 の意図的な読み取り専用方針の反転であり、ファイルシステム書き込み権限の拡大を伴う。書き込み先を `docs/plans/` に限定し、独立レビュアーと self-contained ファイル原則で歯止めとする。
- C-再起動は生成時の暗黙知（設計意図・トレードオフ）の記憶を失う。ただし重要な意図がファイルに無ければ独立レビュアーも理解できないはずで、それは plan の自己完結性の欠陥としてレビューが拾うべきものである。C-継続はそのギャップを私的記憶で覆い隠すため、再起動の方が健全と判断した。
- 命名・修正の責務がメインとエージェントへ再配分されるため、SKILL.md の手順記述の更新が必要になる。

**後続実装 Issue が変更すべき影響範囲**（本 ADR は設計確定のみ。実装は別 Issue）:

- `agents/plan.md`: `allowed-tools` に Write/Edit 追加。行21（読み取り専用の意図）を「plan 成果物の生成＋レビュー駆動改訂の所有」へ改訂。行76（実行ログはスキル側で除去）を「ファイルに載せず返却に含める」へ改訂。編集モード起動の入力契約（パス＋指摘テーブル）を追加。
- `skills/plan-issue/SKILL.md`: step5 で解決済みパスを渡しエージェントが Write。step6 で命名を生成前へ移し Glob 単独・メインの Write/除去を廃止。step7c で FAIL→plan エージェント再起動（メイン Edit を置換）。
- `skills/plan-issue/references/agent-prompt-construction.md`: 編集モード用プロンプトテンプレートの追加、実行ログをファイルに載せない旨の明示。
- `skills/plan-issue/references/split-plan-procedure.md`: 初回モード step2・N回目モード step3-4 の詳細プラン「plan サブエージェントで生成し**メインが Write 保存**」を A（エージェント直接 Write・本文非返却）へ。詳細プランの分割固有命名（`issue-{番号}-1.md`／`-k.md`）への B の適用（メインが Glob で空きパス解決→パス渡し）。詳細プランが step7 自動レビューを経る場合は FAIL 修正を C-再起動へ。分割計画マスター生成（step1、`issue-{番号}-plan.md`）はメイン直接生成のまま据え置く（Decision A 参照）。これを欠くと「一部はエージェント直接 Write・一部はメイン Write」の不整合と未修正の context-budget 違反が詳細プラン経路に残る。

**将来の留保事項**:

- 本設計が崩れる条件は「plan を完全に self-contained にできず、生成時の対話的判断が本質的にファイル化不能」と判明した場合であり、その時は C-継続が正当化される。現行 plan.md は self-contained を要求しており前提は成立している。
- create-issue・refine-issue 1件モードなど ADR-20260627 が挙げた他スキルの本文往復にも同型の責務再配置が適用可能か、後続改修で検討する（本 ADR の射程外）。
