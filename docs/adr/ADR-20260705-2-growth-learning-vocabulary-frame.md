# ADR-20260705-2: growth ループの学習/仮説検証フレームと語彙統一

## Status

Accepted

## Context

#441（spike: 学習/仮説フレームでのパイプライン語彙統一）は、#439（検出器名/型名の最終確定）の命名の詰めから、パイプライン全体の概念フレームの問い直しへ発展した spike である。`DESIGN.md:85` が名指しする「ユビキタス言語移行」epic の起点にあたる。

中核の観察は、growth のループ全体が学習／仮説検証のサイクルとして一貫して読めることにある。promote が既に「予測・反証」（ADR-20260701 の原理2＝反証テスト）で候補を検証している事実が、この読みを裏付ける。捕捉が観測、蒸留が仮説形成（帰納）、promote が仮説検証、検証を生き延びた仮説が「知」である。

本 spike の上流前提は以下の2つの ADR で確定済みであり、本 ADR はそれらを覆さない。

- **ADR-20260701（価値の2軸）**: 判断知＝復元不能性／摩擦知＝再発。型ごとにゲートを一致させる2軸分離。
- **ADR-20260705（サイバネティック設計前提の単一出典）**: 4原理・第二階の警戒・本質変数 open question。特に**能動推論の統一理論を不採用**とし、予測誤差ベースの単一検出器へ畳み込まず、検出器2本を requisite variety として維持する。
  - **検出器名の同定注記**: ADR-20260705 は価値語で「判断知検出子／摩擦検出器」と表記するが、これは #439 が HOW 軸へ確定した **`教示信号検出器` / `予測誤差検出器`** と**同一の検出器2本**を指す（705 は #439 前の価値語で表記）。本 ADR は学習側語彙の単一出典として **#439 の HOW 軸命名を正典**とし、価値語（摩擦／判断）は検出器層でなく価値族＝distill 以降に留保する（#439: WHAT／HOW／価値族の3層分離）。検出方法（HOW）と価値族は直交し、検出器が拾った信号の値の帰属は 1:1 でない（混在ゾーン。#440 が機序を消費）。requisite variety が保つ「2本」は HOW 軸の検出方法2種である。

本 spike が決めるのは上流フレームと、それに従う日本語正典語彙である。#437（型仮説・捕捉段呼称の是正）・#439（`検出子`→`検出器` へ改称・HOW 軸命名）の決定済みだが未反映の語彙決定も本 ADR が巻き取り、**学習側の語彙の単一出典**とする。物理リライト（ファイル横断の語彙書き換え）は #438 が本 ADR を下流で消費して行う。

## Decision

growth ループを学習／仮説検証フレームで読み直し、日本語正典語彙をそのフレームに接地させる。以下を決定する。

### 0. 日本語正典名と英語実装識別子の分離（全決定を貫く原則）

glossary の「見出しは日本語を正典、英語は追跡用の別名」規約に従い、**日本語正典語彙はフレームに合わせて移行し、英語実装識別子（`captures.md`・`candidates.md`・`candidate-status`・英語スキル名 `capture`/`distill`/`promote` 等の起動コマンド・ディレクトリ名）は据え置く**（DESIGN.md:85）。日本語正典が先行し、英語別名・実装識別子の物理変更は #438 以降で追随する。以下 1〜4 はすべて日本語正典の決定である。

### 1. 候補→仮説（単位名）

- distill の出力単位の日本語正典名を `候補` から **`仮説`** へ改める。DESIGN.md 原理2「気付きは仮説であり、検証されるまで配布しない」の instance であり、`仮説`（未検証）→［仮説検証］→ `知`（検証済み）という認識論的ペアが揃う。
- 単位が `仮説` になったことで、既存の scope/career タグの `仮説` 接尾を**外す**。日本語正典名を素の **`スコープ` / `キャリア`**（軸名のみ）とする。暫定性（distill が付与し確証しない／最終裁定は集約点「取り込み Issue」＝ADR-20260628-2）は定義文で与え、ラベルには hedge 語を載せない。
  - `見立て` 等の代替 hedge 語も採らない。容れ物である `仮説` が既に未検証・未確定を宣言しており、内部属性の暫定性は継承で自明であるため、タグごとの hedge 語は冗長である。`provenance`・`status`・`type` 等の他メタ属性が素の名で与えられていることとも一貫する。
- 英語実装識別子 `scope-hypothesis` / `career-hypothesis` は据え置く（原則0）。

### 2. 種別（type）の位置づけ → #440 へ委譲

- 種別（`behavior-diff` / `decision-record`）は仮説（候補）の**内在的な知識型カテゴライズ**であり、ADR-20260701 §4 の知識型読みを維持する。
- 「検証モード」読みは却下する。検証（反証テスト＝原理2）は promote＝蒸留の後工程であり、後工程の関心で仮説の内在属性を命名するのは層の取り違えになる。
- 単一 type（排他）か tag（両型併記）かの partition 硬度は本 spike で確定せず **#440 へ委譲**する（「分類の体系的確度が十分なら type、不十分なら tag」で判断／混在ゾーンの実在と type の暫定性は tag 寄りの材料）。

#### 追補（#440・2026-07-05）: partition 硬度＝非排他 tag で確定

本 decision が #440 へ委譲した partition 硬度を、#440 が同一系譜への in-place 追補として確定する。docs/adr/README.md の「Amended（部分改訂）」手続きは*新規 ADR による旧 ADR の改訂*を想定する（`Amends:`/`Amended by:` の相互併記を要求）が、本件は同 ADR が自ら #440 へ委譲した未決事項を同一 ADR 内で in-place 確定する**自己 Amend 形**であり、新規 ADR を起票しないため README 手続きに完全準拠はしない（`Amends:`/`Amended by:` の代わりに委譲先 Issue #440 への逆参照で対応する）。Status は Accepted 維持。

- **確定値**: **非排他 tag（多値 set）**。排他 type を廃し、`behavior-diff` / `decision-record` を tag 値として存続させる（廃止されるのは排他制約であってカテゴリ自体ではない）。`type` は tag の要素数1特殊ケースであり **tag が type を包含する**（確信ケースは要素数1で表せるため type を別途持たない）。本 decision の裁定基準「不十分なら tag」の tag 寄り示唆を tag 確定で充足する。
- **仮説形成（distill）の分岐**: observation 段の型仮説（#437）を distill が **evidence-gated** で確定する（既定は単一タグ、第2タグは陽性証拠がある時のみ付与し既定 both を禁ずる）。ADR-20260701 §4「distill 出力2系統分離」は不変（系統数2のまま、両タグ候補が両系統に入り系統メンバーシップが関数から集合関係になるのみ）。
- **`candidates.md` の表現**: `type`（単値）→ `tags`（多値 set、値域 {behavior-diff, decision-record} の部分集合）へ改称する設計決定。物理スキーマ改称の実装は #438／後続実装 Issue（OUT）。
- **一次記録**: `plugins/growth/DESIGN.md` §6 決定事項10（母艦）。代表パターン・二重損失回避・requisite variety との整合確認・軸区別（知識型軸 vs 出所軸）を含む。

### 3. 型名（判断知/摩擦知） → 制約記録に collapse

- 型名（`振る舞い差分` / `文脈付き決定知`）は確定済みの価値2軸区別（ADR-20260701 §1）に貼るラベルであり、本 spike で新たな判断はしない。
- 本 spike が残す制約は2点: ①「知＝検証後」（仮説段階のラベルは「知」を段階インデックスとして主張しない）②glossary 正典規約（見出しは日本語を正典）。最終ワーディングは #440 の type/tag 構造確定後に #438 の呼称リライトで当てる。

### 4. 段の日本語正典名を学習語彙へ移行

- 中核3段の日本語正典名を、機構メタファから学習語彙へ移行する。

| 現行（機構名） | 学習語彙（正典） |
|---|---|
| 捕捉（capture） | **観測** |
| 蒸留（distill） | **仮説形成**（方法の注記＝帰納） |
| 昇格（promote） | **仮説検証** |

- `配布`・`測定 / 撤回` は学習層でなくサイバネティクス層（ADR-20260705）に属するため学習語彙化せず据え置く。
- 統合像（認識論の弧）: `観測 →［仮説形成/帰納］→ 仮説 →［仮説検証］→ 知`。仮説形成が仮説を生み、仮説検証が仮説を知に変える。
- 英語スキル識別子・ディレクトリ名（`capture`/`distill`/`promote`）は据え置く（原則0）。

### 5. ADR-20260701 との関係 → Related 補完（Amend / Supersede いずれも不要）

- 本 ADR は ADR-20260701 の**どの facet も改訂しない**。価値2軸・type の2系統分離・型適応検証はすべて追認する。decision 1 が退役させる「型仮説」の語は #437 由来（`capture/SKILL.md`）であって ADR-20260701 の語ではなく、decision 2 は §4 の知識型読みを追認する。動くのは日本語正典語彙のみで、**語彙移行は決定改訂ではない**。
- したがって Amend でも Supersede でもなく、**Related 補完**とする。ADR-20260705 が同じ状況（2軸の理論的基礎づけを加えるが決定本文は変えない）で採った扱いと同型である。発見可能性のため双方の「関連ADR」節に `Related` を併記する。

## Consequences

- **得られる利益**: growth ループが `観測 → 仮説形成 → 仮説 → 仮説検証 → 知` の一貫した認識論の弧として読め、日本語正典語彙が学習/仮説検証フレームに接地する。#437/#439 の未反映語彙決定も本 ADR に集約され、学習側語彙の単一出典が立つ。

- **能動推論の統一理論へ畳み戻さないことの確認（#441 AC6）**: `仮説` は behavior-diff と decision-record の両型を覆う総称ラベルにすぎず、検出器2本（予測誤差検出器／教示信号検出器）・型2系統（behavior-diff／decision-record）・検証2モード（予測反証／復元不能性）はいずれも不変である。ADR-20260705 の能動推論不採用の懸念は検出器層の問題であって単位名の問題ではなく、`仮説` 命名は「予測誤差の単一統一検出器」への畳み戻しを含意しない。requisite variety（検出器2本維持）は保たれる。

- **受容したコスト**: 日本語正典と英語実装識別子の間に一時的な語彙ラグが生じる（正典 `仮説`/`スコープ`/`キャリア` に対し実装は `candidates.md`/`scope-hypothesis`/`career-hypothesis` のまま）。このラグは原則0で意図的に許容し、#438 以降の物理リライトで追随する。

- **下流へ据え置く範囲**:
  - **#440**: 種別の partition 硬度（単一 type / 両型併記）の機序。→ **#440 で確定済み（非排他 tag。上記 decision 2 の追補を参照）**。
  - **#438**: 本 ADR の確定語彙のファイル横断反映。#438 のスコープは #437 原案（摩擦シグナル／判断シグナル＝型仮説）から本 ADR モデル（観測／学習シグナル＋予測誤差・教示信号検出器／仮説／仮説形成・仮説検証／スコープ・キャリア）へ差し替える。
  - **scope/career の外来語→訳語化**（`スコープ`→適用範囲 等）: DESIGN.md:85 の epic に列挙される別軸の変更。本 spike のスコープ外とし、別 follow-up で扱う。

- **ADR-20260701 との関係**: 本 ADR は Supersede でも Amend でもなく Related 補完である。ADR-20260701 の Status（Accepted）・本文の決定は変更しない。
