# ADR-20260525: サブエージェント起動時の CLAUDE.md 取得方針

## Status

Accepted

## Context

共通スキル（`plan-issue` 等）が `Agent` ツールでサブエージェントを起動する際、プロンプトに「ユーザースコープ / プロジェクトルートの CLAUDE.md を読み、そのルールに従うこと」という指示を含めていた。

この指示によりサブエージェントは起動直後に `Read` / `Bash(cat ~/.claude/CLAUDE.md)` を実行する。`~/.claude/CLAUDE.md` は作業ディレクトリ外のため、Claude Code の権限モデルにより毎回ツール許可プロンプトが発生し、dev-loop 等のワークフロー実行を中断させていた（#216）。

実機検証により、サブエージェント（`Agent` ツール）にもユーザースコープ / プロジェクトルートの CLAUDE.md がコンテキスト冒頭の `<system-reminder>` 内 `# claudeMd` セクションへ自動注入されることを確認した。Read / Bash / Grep を一切禁止した制約下でも、サブエージェントは両 CLAUDE.md の全項目を該当箇所引用付きで回答でき、ツール呼び出しはゼロ件だった。したがって明示的な読み取り指示は完全に冗長である。

## Decision

共通スキルから起動するサブエージェントには、CLAUDE.md の明示的な読み取り（`Read` / `Bash(cat ...)`）を指示しない。サブエージェントへ自動注入されるコンテキストに依存する。

- スキル定義・サブエージェント定義・プロンプトテンプレートに「CLAUDE.md を読め」という指示を記述しない
- 要約してプロンプトへインライン展開する対応も不要（自動注入で足りるため）

## Consequences

- 許可プロンプトの頻発が解消され、ワークフローが中断されなくなる
- 共通スキルがユーザー固有ファイル（`~/.claude/CLAUDE.md`）への明示アクセスに依存しなくなり、再利用性が向上する
- 自動注入の挙動は Claude Code 側の仕様に依存する。将来この挙動が変わった場合、サブエージェントが CLAUDE.md 規約を参照できなくなるリスクがある。その時点で本 ADR を再検討する
- ユーザースコープ `~/.claude/CLAUDE.md` の「サブエージェントの実行記録」セクションに同種の規約が残存しており、本リポジトリ外のため別途撤回が必要（撤回しない限り再発リスクが残る）
