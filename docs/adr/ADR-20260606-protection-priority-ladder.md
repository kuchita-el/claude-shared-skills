# ADR-20260606: 保護手段の優先（2軸マップとコスト×価値ゲート）

## Status

Accepted

## Context

`docs/principles.md` は「ドメイン構造を先に作る（70%の見通し）」までは原則化していたが、**保護手段の優先順位**——「違反が書けない形にできるなら指示は不要」という、構造から指示までの優先順位——を明文化していなかった。このため、何かを守りたいときに安易に CLAUDE.md/Skill への指示追加に倒れやすく、指示は確率的にしか効かないうえ肥大化しやすいという課題があった。

原則④「違反を書けない構造を、指示より優先する」を principles.md 基本原則に追加するにあたり、その運用枠組み（保護手段をどう位置づけ、どの段で守るかをどう判断するか）を確定する必要が生じた。設計過程で複数の表現案・判別子案を検討し、一部を却下した。

関連Issue: #228（本原則の起票）、#157（DDDドキュメントの機械可読化＝本原則の適用先）、#229（削除方向＝棚卸し基準の追加）

## Decision

1. **保護手段を「決定論性 × 持続性」の2軸マップで表現する。** 当初検討した線形ラダー（`構造 > 型 > 依存方向 > scaffolding > テスト > 指示`）は採用しない。scaffolding とテストが同一軸に乗らない（前者は一度きり、後者は継続）ため、線形順序が誤読を生む。

2. **scaffolding を二分する。** ワンショット型（テンプレ・ジェネレータ＝生成時のみ、ドリフトを止められず弱い）と、継続生成型（スキーマからの codegen、手編集禁止で再生成が継続＝ドリフトせず型と同格）を区別する。

3. **段の選択はコスト×価値ゲートで判断する。** ゲートは非対称構造とする:
   - 主軸＝**検出性**（すり抜けるか）。沈黙する違反は上位段を選好、loud に落ちる違反はテスト・指示で足りる。
   - **不可逆性**を無条件昇格の**オーバーライド**として組む（対称な第2軸にはしない）。回復不能なら検出性・コストに関わらず上位段へ。
   - コスト側の補助線として**反復性**を添える（多数箇所で繰り返す違反は構造化が償却され上位段が割安）。

4. **指示段へ落とす際の理由は、対象コードの近傍コメントに1行残す。** PR説明・ADR は却下（近傍コメントが再生成・移動に追従し継続的に検証可能なため）。

5. **二層構造をとる。** 原則の内容（ストック）は principles.md 本体（原則④＋「保護手段の優先」節）に置き、本ADRは採用決定と却下根拠を記録する。自律度ラダー（principles.md 本体＋ADR-20260602-2）と同じパターン。

6. **削除方向（棚卸し）を同枠組みの対称物として含める（#229）。** コスト×価値ゲートが指示段へ「落とす」判断なら、その対称として既存指示を「外す」判断（棚卸しの3パターン＝観点を絞る／層を移す／目的を失った指示を外す）も同一原則の表裏である。独立節を新設せず principles.md「保護手段の優先」節に統合し、指示の追加と削除を一箇所で完結させる（論理分散を避けるため）。

### 却下した案（採用理由の揮発防止）

| 却下案 | 却下理由 |
|---|---|
| 線形6段ラダー | scaffolding とテストが同一軸に乗らない。一度きり／継続の差を線形が潰す |
| 検出性 × 不可逆性 の対称2×2 | 不可逆性は滑らかな次元でなく不連続（損害が無限大になる一点）。対称軸化は誤モデル化であり、かつ「可逆だが甚大（blast radius 大）」を取りこぼす |
| severity（損害）を第2軸にする | 二値テストが消えてアドホック判断が戻る。ゲート自身が肥大化し、ゲートのコスト×価値原則にメタレベルで反する |
| 永続性を軸にする | 不可逆性に吸収される。単独では弱い（cosmetic な違反も永続しうるが回復は一瞬） |
| 集約境界またぎを判別子に追加 | 採用アーキテクチャ依存で普遍性がない。沈黙×不可逆が最悪ケース（知識汚染ループ）を既に捕捉する |

## Consequences

**得られた利益:**

- 決定論的保護を選好する判断が機械化され、指示肥大化を抑制できる。
- 「検証ギャップ」原則の構成的な裏返し（人が検証しきれないなら検証を要さない手段を優先）として、既存原則と一貫する。
- #157（機械可読化）に根拠を与える——規約（指示）を構造・型へ昇格させる試みは本原則の適用実例。

**受容したコスト・トレードオフ:**

- 2軸マップは線形ラダーの素朴な分かりやすさを失う。
- severity 軸を入れないため、「可逆だが甚大」なケースは検出性軸のみで扱う。不可逆性オーバーライドで最悪ケースを押さえた上での意図的な割り切り。
- ゲートに残るアドホック判断面（曖昧な中間セル）は完全には消えない。判断を全廃せず、危険が明白なケースだけ機械化し、曖昧なケースに判断を温存する設計。
- 2軸マップのセル配置は固定ではない。「依存方向」「継続生成codegen」の決定論性は強制手段（lint/CI 実行時チェック か、モジュール機構・read-only 等の構造的強制か）に依存して象限が変わる。マップはこのドリフトを可視化する道具であり、配置を一意の正解として読まない。

**将来の留保:**

- 反復性・継続生成codegen の扱いは運用を通じて見直す余地がある。
- #157 実装時に、本原則との整合（規約のどこを構造・型へ昇格させるか）を再確認する。

**粒度判定記録（4項目チェックリスト）:** 後戻りコスト高・複数モジュール/開発者に波及・採用理由が揮発しやすい・ツールで自動強制できない、の4項目すべてに該当（4点）。3点以上のため ADR 化。
