# ADR-20260601: 自律度の表現を責任分担マトリクスの承認ゲート軸に置き、ドメインモデルへ持ち込まない（ADR-20260421整合）

## Status

Accepted

## Context

自律度モデル（L0–L3）の導入が親トラッカー Issue #232 で進行している。導入にあたり、自律度（特に L2/L3 の承認ゲート有無の差）を **どこに・どう表現するか** を確定する必要がある。

先行する ADR-20260421 は「コーディングエージェント固有挙動をドメインモデルに持ち込まず、アクター内部実装として抽象化する」を確定済みである。自律度を Command の属性などとしてドメインモデルへ持ち込むと、この決定と衝突する懸念がある。

Issue #232 の確定設計判断①で **非重複形** の採用が決定された。本 ADR はその整合方式（自律度をドメインモデルへ持ち込まず、責任分担マトリクスの軸として表現する）を Accepted として固定し、後続の規約 ADR（#238）・判定ルーブリック（#239）・記法拡張（#240）のブロッカーを解除する位置づけにある。

## Decision

1. **非重複形を採用する**。自律度を Command の属性として追加せず、ドメインモデル（`docs/development/event-storming.md`）には自律度パラメータを持ち込まない。これは ADR-20260421 の「エージェント固有の制御パラメータをモデルに持ち込まない」決定と同方向であり、これを補強する。
2. **承認ゲート軸で表現する**。L2/L3 の差（承認ゲートの有無）は `docs/principles.md` の責任分担マトリクスへ **承認ゲート軸** を追加して表現する。すなわち Explanation 層（根拠ハブ）で表現し、ドメイン構造層とは層を分ける。
3. **本 ADR のスコープは整合方式の確定までとする**。L0–L3 各レベルの正式定義（名称・判定基準）、判定ルーブリック、ゲート二段表現の記法拡張は本 ADR では扱わない。これらは規約 ADR（#238）・判定ルーブリック（#239）・記法拡張（#240）の責務とする。
4. **ADR-20260421 との非重複性**。ADR-20260421 はドメインモデル層の決定（モデルに持ち込まない）、本 ADR は Explanation 層での表現の決定（責任分担マトリクスの承認ゲート軸で表現する）であり、対象とする層が異なる。両者の決定範囲は重複せず、本 ADR は ADR-20260421 を改訂・上書きしない。よって関係種別は Related とする。

## Consequences

**得られた利益**:

- ドメインモデルを変更せずに自律度の差（承認ゲート有無）を表現でき、ADR-20260421 と矛盾しない。
- 責務が層で分離される（ドメイン構造層＝モデルに持ち込まない／Explanation 層＝責任分担マトリクスで表現）。
- 承認ゲートの二段表現（提案→承認）は技術的意思決定集約のライフサイクル（Proposed→Accepted、ADR-20260511）と同型であり、既存パターンの再利用で表現できる。

**受容したトレードオフ**:

- 自律度の表現が Explanation 文書（責任分担マトリクス）に置かれるため、ドメイン構造層からは直接見えない。横断的な可視性は中程度にとどまる。

**将来の留保事項**:

- L0–L3 各レベルの正式定義（名称・判定基準）は規約 ADR（#238）で確定する。
- 判定ルーブリック（自律度を決める5要因と ADR 粒度判定の発想上の対応を含む）は #239 で作成する。
- ゲート二段表現の記法拡張は #240 で扱う（規約の中身が固まった後に着手）。
