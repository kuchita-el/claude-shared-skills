# ADR-20260606-2: 指示の棚卸し（保護手段の優先の削除方向）

## Status

Accepted

## Context

[ADR-20260606](ADR-20260606-protection-priority-ladder.md)（保護手段の優先）は、コスト×価値ゲートで「いつ守りを指示段へ**落とすか**」＝追加方向の判断を確定した。一方、その対称となる「いつ既存の指示を**外すか**」＝削除方向の判断基準は欠落していた。

指示（CLAUDE.md/Skill）は追加が容易で、削除の判断基準を欠くと放置される。読まれない指示が溜まり、モデル更新時に「どれが今も有効か」を判断できずドリフトする。原則④「違反を書けない構造を、指示より優先する」を運用しきるには、追加側だけでなく削除側のゲートも要る。

本ADRは ADR-20260606 が確定した枠組みに削除方向の決定を追加する（部分改訂＝Amends）。ADR-20260606 の既存5決定は不変。

関連Issue: #229（棚卸し基準の起票）

## Decision

1. **削除方向（棚卸し）を「保護手段の優先」枠組みの対称物として含める。** コスト×価値ゲート（追加方向）と棚卸し（削除方向）は同一原則の表裏であり、「いつ指示を足し、いつ外すか」を一体で扱う。

2. **棚卸しの観点を3パターンで定義する。** (a) 観点を絞る（低優先観点を落とし重要観点に集中）、(b) 層を移す（上位段または自動レビューAgentへ移し指示から削除）、(c) 目的を失った指示を外す（判定基準=「想定する違反は今も起こりうるか／起きたとき問題か」、両方Noで削除可）。

3. **削除方向は principles.md「保護手段の優先」節へ統合する。独立節は新設しない。** 追加方向（コスト×価値ゲート）と削除方向（棚卸し）を一箇所で完結させ、論理の分散を避ける。二層構造は ADR-20260606 を踏襲（原則の内容＝principles.md 本体のストック、採用決定＝本ADRのフロー記録）。

### 却下した案（採用理由の揮発防止）

| 却下案 | 却下理由 |
|---|---|
| principles.md に独立した新節を設ける | 追加方向と削除方向が2箇所に分散し、「保護手段の優先」参照時にどちらを指すか曖昧化する。同一原則の表裏を分断する |
| ADR-20260606 を全体 Superseded して書き直す | 既存5決定は有効なまま。全体無効化は過剰で、当時の README の Superseded/Amended 判別基準（旧ADRの大半を無効化するかで選ぶ）上も Amended が正しい（Amended 機構はその後 ADR-20260711-3 決定4 で廃止） |
| ADR-20260606 本文に in-place 追記する | Accepted 後の ADR 決定本文を不変として扱う本リポの運用（新ADR＋相互参照で記録）に反する |

## Consequences

**得られた利益:**

- 指示の追加と削除が同一ゲートで判断でき、指示肥大化・モデル更新時のドリフトを抑制できる。
- ADR-20260606 と合わせ、保護手段の優先が追加・削除の双方向で完結する。

**受容したコスト・トレードオフ:**

- 棚卸しパターン (c) の判定文（AND 論理）は、パターン3スコープの残余ケース向け。グローバルな削除ゲートと誤読されるとパターン1/2が拾うケースを取りこぼす印象を与えうるため、principles.md 側で適用範囲を明示する。
- 当時の README の「Amended」手続きは語義上「既存facetの改訂」を対象とし、本ADRの「新facetの追加」に厳密対応する手続きは存在しなかった。最も近い Amended（新ADR＋相互参照＋旧本文不改変）を準用した（Amended 機構はその後 ADR-20260711-3 決定4 で廃止。現行では非core改訂＝直接編集＋`## 変更履歴` が相当する）。

**粒度判定記録（4項目チェックリスト）:** 後戻りコスト=中（文書追記・可逆）、波及=単一文書、採用理由の揮発性=中（独立節却下・全体Superseded却下の判断が揮発しうる）、ツール自動強制=不可。単独では3点未満だが、ADR-20260606 の枠組みを直接改訂し、かつ却下根拠の揮発防止価値が高いため、親ADRの Amended として記録する。
