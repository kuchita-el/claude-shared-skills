# ADR-20260415: DDDドキュメント整合性レビューでの責務境界考慮

## Status

Accepted

## Context

PR #124（`refactor: Big Pictureを7コンテキストに再設計し3語体系を採用`、MERGED 2026-04-15）のレビューで、Big Picture ホットスポット #7 解消アクションが QA ドキュメントを参照していたが、QA 側に「差し戻し判断」の記述がないことを不整合（Critical）と判定した。

実際にはこの判定は誤りで、以下の2点を区別していなかったことが原因だった。

1. **リンクの意味の混同**: 構造的参照（QA設計更新の結果として BP#7 が解消された、という結果参照）と、個別記述参照（対応する記述が存在する、という内容参照）を区別していなかった
2. **責務境界の誤認**: DDD コンテキスト境界では、責務外事項は記述しないことが正しい設計。QA は「差し戻し判断」を持たない設計のため、QA ドキュメントに該当記述がないのは整合の証拠（境界が正しく引かれていることを示す）であり、不整合ではない

加えて、「差し戻し判断」（BL 側のトリアージ）と「バグ起票」（QA 側のイベント発火）を同一レイヤーで扱う概念混同もあった。

## Decision

DDDドキュメント整合性レビュー時、ドキュメント間相互参照の「参照先記述欠落」を検知した場合、即 Critical 判定せず、以下の2段階チェックを行う。

1. **リンクの意味を識別する**
    - 構造的参照: 「設計更新の結果として X が解消された」のような結果参照。参照先の個別記述の存在は要求しない
    - 個別記述参照: 「対応する記述が存在する」のような内容参照。参照先の該当記述の存在を期待する
2. **記述欠落の理由を判定する**
    - 責務境界による正常な省略: 参照先コンテキストの責務範囲外であれば、記述がないのが整合の証拠
    - 実際の記述漏れ: 責務範囲内なのに記述がなければ不整合

レビュー観点プロンプトに「参照=個別記述対応」前提を埋め込まないこと。

## Consequences

**得られた利益**:

- DDD コンテキスト境界による正常な省略を誤って Critical 判定するレビューノイズが減る
- 責務境界が正しく引かれていることを「整合の証拠」として認識できる
- レビュー観点プロンプトの設計品質（観点4の文言設計）への改善示唆を提供

**受容したトレードオフ**:

- レビュー観点が「参照=個別記述対応」の単純な機械的判定よりも複雑になり、レビュアーの DDD コンテキスト境界の理解が要求される
- 自動lintで完全に判定するのは困難。`scripts/lint-domain-doc.sh` の対象範囲を超える領域での運用判断に依存

**将来の留保事項**:

- review-guide-default.md（観点#7の文言設計、Issue #112 で見直し済）など、レビューガイド側への反映は別途検討
- DDD ドキュメントの構造的参照と個別記述参照を文法上区別する記法（記法ガイド側）は未検討
