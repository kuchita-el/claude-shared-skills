# イベントストーミング（Event Storming）

イベントストーミング手法に基づいて対話的にドメインを分析し、ドキュメントを生成する。
Big Picture（システム全体俯瞰）とDesign Level（単一ドメイン深掘り）の2つのフェーズに対応する。

## 引数

- `$ARGUMENTS` の解析ルール:
  - **引数なし** → Big Pictureから開始
  - **ドメイン名あり** → Design Levelから直接開始
    - 例: `order`, `在庫管理`, `注文 ECサイトの注文ドメイン`
    - ドメイン名はディレクトリ名としても使用する
  - `--auto` のみ → Big Picture + Autoモード
  - `ドメイン名 --auto` → Design Level + Autoモード

## モード

| モード | 起動条件 | 動作 |
|---|---|---|
| Interactive | デフォルト | 各フェーズでユーザー確認を挟んでから次に進む |
| Auto | `--auto` または「一気に」「全部出して」等 | 入力を受け取った後、全フェーズを一括実行する |

**Autoモードの入力:**

- Big Picture: システム概要
- Design Level: ドメイン概要

入力なしでは的外れな出力になるため、概要入力は省略しない。

**Autoモードの動作ルール:**

- 各フェーズの「ユーザーに確認する」「修正を求める」等の対話ステップはスキップし、AIの判断で次フェーズに進む
- 概要をもとにAIが各フェーズの成果物を自律的に作成する
- 全フェーズ完了後にまとめてドキュメントを生成し、最終確認のみユーザーに求める

## フェーズ別フロー

### Big Picture（引数なしで起動）

システム全体を俯瞰し、ドメインイベントの洗い出し → イベントフロー整理 → バウンデッドコンテキスト発見 → コンテキスト間の依存方向整理を行う。

**詳細手順:** `${CLAUDE_SKILL_DIR}/references/big-picture-flow.md` を参照

**出力:** `docs/big-picture.md`

### Design Level（ドメイン名指定で起動）

単一ドメインを深掘りし、イベント・コマンド・アクター・集約・状態遷移を詳細に分析する。

**詳細手順:** `${CLAUDE_SKILL_DIR}/references/design-level-flow.md` を参照

**出力:** `docs/{domain}/event-storming.md`

## 対話の原則

- **ユーザーの言葉を尊重する**: ドメイン用語はユーザーが使った表現をそのまま採用する。勝手に一般化・抽象化しない
- **段階的に進める**: 各フェーズの結果をユーザーに確認してから次に進む。一度に全フェーズを実行しない（Autoモード除く）
- **具体例で導く**: 抽象的な質問だけでなく、ドメインに合わせた具体例を挙げて理解を助ける
- **過剰な抽象化を避ける**: 小規模ドメインに不必要な複雑さを持ち込まない

## Big Picture ↔ Design Level の整合性ルール

- **Design Level実施時**: `docs/big-picture.md` が存在する場合、そのコンテキスト境界を尊重する（矛盾する集約の分割やイベントの再定義をしない）
- **Big Picture実施時**: `docs/big-picture.md` を更新した場合、影響を受けるコンテキストの既存Design Level文書（`docs/{domain}/event-storming.md`）を確認し、修正が必要な箇所をユーザーに案内する

## 注意事項

- **フレームワーク非依存**: 出力はドメインの設計ドキュメントであり、特定の言語・フレームワーク・ORMに依存しない
- **Mermaid記法**: 図にはMermaid記法を使用する（GitHub/GitLab等で直接レンダリング可能）
