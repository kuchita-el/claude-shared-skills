# ADR作成スキル

ADR（Architecture Decision Record）の作成を対話形式で支援します。

## 手順

### 1. 既存ADR確認・採番

プロジェクトのADRディレクトリ（慣習的に `docs/adr/`）をスキャンし、最大番号 + 1 を次の番号とする（4桁ゼロ埋め）。

```bash
ls docs/adr/*.md
```

例: `0001-error-handling-design.md` が存在する場合、次は `0002`

### 2. 基本情報収集

ユーザーに以下を質問する:

1. **タイトル**: 技術的決定の簡潔な説明（例: 「エラーハンドリング設計」「認証方式の選定」）
2. **コンテキスト**: この決定が必要になった背景・課題

### 3. 詳細情報収集

ユーザーに以下を質問する:

1. **決定内容**: 何を決定したか（具体的な技術選定、設計方針など）
2. **理由**: なぜこの決定をしたか（他の選択肢との比較、トレードオフの考慮など）
3. **結果**: この決定による影響・トレードオフ（メリット・デメリット）

### 4. 関連ADR探索

入力されたタイトルやコンテキストからキーワードを抽出し、既存ADRを検索する。

```
Grep で docs/adr/ 内を検索
```

関連しそうなADRが見つかった場合、リンク追加を提案する。

### 5. ステータス確認

ユーザーに以下を質問する:

- **ステータス**: 以下から選択
  - `Proposed`（提案中）- デフォルト
  - `Accepted`（採用）
  - `Deprecated`（非推奨）
  - `Superseded`（置き換え済み）

`Superseded` が選択された場合:

- 後継ADRの番号を質問する

### 6. プレビュー・確認

生成するADRの内容をプレビュー表示し、修正の機会を提供する。

```markdown
---プレビュー---

# ADR-{NNNN}: {タイトル}

## ステータス

{ステータス}

## コンテキスト

{背景・課題の説明}

## 決定

{決定内容}

## 理由

{なぜこの決定をしたか}

## 結果

{この決定による影響・トレードオフ}

## 関連

{関連ADRリンク（あれば）}

---

プレビュー終わり---
```

ユーザーに確認:

- この内容でよいか？
- 修正したい箇所はあるか？

### 7. ADRファイル生成

確認が取れたら、以下のパスにファイルを生成する:

```
docs/adr/{NNNN}-{タイトル（ケバブケース）}.md
```

タイトルのケバブケース変換ルール:

- スペースをハイフンに置換
- 日本語はローマ字に変換（例: 「エラーハンドリング設計」→ `error-handling-design`）
- 小文字に統一

## ADRテンプレート

```markdown
# ADR-{NNNN}: {タイトル}

## ステータス

{ステータス}

## コンテキスト

{背景・課題の説明}

## 決定

{決定内容}

## 理由

{なぜこの決定をしたか}

## 結果

{この決定による影響・トレードオフ}

## 関連

{関連ADRリンク（任意）}
```

## ステータスの書き方

| 選択値     | ADRへの記載                                             |
| ---------- | ------------------------------------------------------- |
| Proposed   | 提案中（Proposed）                                      |
| Accepted   | 採用（Accepted）                                        |
| Deprecated | 非推奨（Deprecated）                                    |
| Superseded | 置き換え済み（Superseded by [ADR-XXXX](./XXXX-xxx.md)） |

## 完了メッセージ

ファイル生成後、以下を出力:

```
ADRを作成しました: docs/adr/{NNNN}-{ファイル名}.md

次のステップ:
- 内容を確認し、必要に応じて編集してください
- チームメンバーにレビューを依頼してください
- 承認後、ステータスを「採用（Accepted）」に更新してください
```

## 注意事項

- 対話形式で情報を収集し、ユーザーの負担を軽減する
- 既存ADRとの関連性を自動で探索し、リンク追加を提案する
- プレビューで確認の機会を設け、修正を受け付ける
- ファイル名は英語のケバブケースで統一する
