# 優先度の高いIssueをピックアップ

GitHubのオープンIssueを分析し、優先度の高いものをピックアップして提案します。

## 手順

### 1. オープンIssueの一覧取得

```bash
gh issue list --state open --limit 20 --json number,title,labels,milestone,createdAt,updatedAt
```

### 2. 主要Issueの詳細確認

各Issueの本文を確認し、以下の情報を抽出する:

- 優先度の明記（本文中の「優先度: Low/Medium/High」など）
- 前提条件・依存関係
- 影響範囲（ユーザー体験、アーキテクチャ、コード品質など）
- 作業規模

```bash
gh issue view {number} --json body,title
```

### 3. 優先度の判断基準

以下の基準で優先度を判断する:

1. **本文中の優先度明記**: Issue本文で優先度が明記されている場合はそれを尊重
2. **ビジネス影響度**: ユーザー体験に直結するもの > 内部品質改善
3. **技術的依存関係**: 他のタスクの前提条件となるもの > 単独タスク
4. **作業規模**: 小規模で着手しやすいもの > 大規模リファクタリング
5. **緊急性**: 本番運用に必要 > 将来対応でよいもの

### 4. 出力フォーマット

以下の形式で優先度別にIssueを提示する:

```markdown
## 優先度の高いIssue

### 🔴 高優先度

| #       | タイトル | 理由       |
| ------- | -------- | ---------- |
| **#XX** | タイトル | 理由の説明 |

### 🟡 中優先度

| #       | タイトル | 理由       |
| ------- | -------- | ---------- |
| **#XX** | タイトル | 理由の説明 |

### 🟢 低優先度

| #       | タイトル | 備考 |
| ------- | -------- | ---- |
| **#XX** | タイトル | 備考 |

---

**推奨**: 次に着手すべきIssueの提案
```

## 補足

- 現在作業中のブランチ名からIssue番号を推測し、作業中のものは除外する
- クローズ済みのIssueは対象外
- ラベルやマイルストーンが設定されている場合は優先度判断の参考にする
