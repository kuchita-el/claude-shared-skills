# PRレビュー指摘フィックスパイプライン

PRレビューの指摘を修正し、テスト・lint・差分確認・コミットまでを一貫して実行する。再レビューで新たな指摘があれば自動で修正ループを継続する。

## 引数

- `$ARGUMENTS`: PR番号またはURL（省略可。例: `123` または `https://github.com/owner/repo/pull/123`）

## 前提条件

- PRが作成済みであること
- 以下のいずれかでレビュー指摘が取得可能であること:
  - 会話にレビュー指摘が含まれている（レビューツールの実行結果等）
  - PR番号が指定されている（GitHubから直接取得）

## ワークフロー

```
外側ループ（最大3周）:
  Phase 0: 指摘の整理
  Phase 1: 指摘ごとの修正
  Phase 2: 再レビュー
  Phase 3: 判定 → 指摘なし→完了報告 / 指摘あり→Phase 0に戻る / 3周目→打ち切り報告
```

### Phase 0: 指摘の整理

以下の優先順位で指摘を取得する:

1. **会話コンテキスト**: 直前のレビュー結果が会話に含まれていればそれを使用
2. **GitHubから直接取得**: PR番号が指定されている場合、`gh pr view` でレビューコメントを取得

取得した指摘を「修正必須」（バグ、セキュリティ、規約違反）と「提案」（リファクタリング、命名改善等）に分類し、ユーザーに対応する指摘を確認する。

### Phase 1: 指摘ごとの修正

確認済みの指摘を1件ずつ修正する。関連する指摘はまとめて修正してよい。

各指摘について以下を実行する:

1. **コード修正**: 影響範囲を確認してから修正する
2. **テスト実行**: 失敗時は原因を分析して修正（最大3回リトライ、解決しなければユーザーに報告）
3. **Lint実行**: 自動修正を試み、それでも直らなければ手動修正（最大3回リトライ）
4. **差分確認・ステージング**: 変更が意図通りか確認し、対象ファイルのみをステージングする（`git add -A` は使わない）
5. **コミット**: テスト・lintが全てパスした場合のみコミットする。修正内容がわかるメッセージを作成し、対応した指摘を本文に記載する

テスト・lintコマンドの探索優先順位:
1. `CLAUDE.md` の記載
2. `package.json` の scripts
3. `Makefile` のターゲット
4. 見つからない場合はユーザーに確認

### Phase 2: 再レビュー

利用可能なレビューツール（`/pr-review-toolkit:review-pr` 等）があれば実行する。なければ修正した全ファイルを読み直し、指摘の意図が正しく反映されているか・新たなバグが混入していないかをセルフレビューする。

### Phase 3: 判定

- **指摘なし** → 完了報告
- **指摘あり かつ 3周目未満** → Phase 0に戻る
- **3周目** → 打ち切り報告

### 完了報告

PRに対応サマリをコメント投稿する。フォーマット:

```markdown
## レビュー指摘対応完了

全ての指摘に対応しました。

### 対応サマリ

| 指摘 | 対応内容 | コミット |
| ---- | -------- | -------- |
| ...  | ...      | ...      |

---

_自動投稿: Claude Code (`/fix-review`)_
```

### 打ち切り報告

解消できなかった指摘がある場合、対応済み・未解消の一覧をPRにコメント投稿する。フォーマット:

```markdown
## レビュー指摘対応（一部未解消）

3回の修正ループで解消できなかった指摘があります。手動確認をお願いします。

### 対応済み

| 指摘 | 対応内容 | コミット |
| ---- | -------- | -------- |
| ...  | ...      | ...      |

### 未解消

| 指摘 | 未解消理由 |
| ---- | ---------- |
| ...  | ...        |

---

_自動投稿: Claude Code (`/fix-review`)_
```

## 注意事項

- **テスト・lint・pushコマンドの実行許可**: プロジェクトごとに異なるため `allowed-tools` には含めていない。`settings.json` の `allowedTools` でプロジェクトごとに許可設定すること（例: `Bash(npm test*)`, `Bash(npm run lint*)`, `Bash(git push*)`）
- **PRコメント投稿**: 複数行のコメントは Write ツールで一時ファイル（`/tmp/fix-review-{timestamp}.md`）に書き出してから `gh pr comment --body-file` で投稿する（allowed-tools の制約回避）
