# Definition of Ready (DoR)

Issueが作業開始可能かどうかを判定するための基準です。

## サイズ別チェック項目

### Small

**対象**: バグ修正、typo修正、軽微な変更、単純な設定変更など

| 項目             | 説明                                   |
| ---------------- | -------------------------------------- |
| 課題が明確       | 何を解決するのかが明確に記載されている |
| 完了条件がわかる | 何をもって完了とするかが判断できる     |

### Medium

**対象**: 機能追加、既存機能の改善、リファクタリングなど

Small のチェック項目に加えて:

| 項目                               | 説明                                       |
| ---------------------------------- | ------------------------------------------ |
| 背景・目的が明確                   | なぜこの変更が必要なのかが説明されている   |
| 要件が具体的                       | 実装すべき内容が具体的に記載されている     |
| 受け入れ条件（AC）が定義されている | テスト可能な形式で完了条件が列挙されている |
| 見積もり可能                       | 1PR（1-2日）程度で完了できる規模である     |
| 不明点がクリアになっている         | 曖昧な表現や複数解釈可能な箇所がない       |

### Large

**対象**: アーキテクチャ変更、大規模機能追加、設計変更など

Medium のチェック項目に加えて:

| 項目                         | 説明                                           |
| ---------------------------- | ---------------------------------------------- |
| トレードオフが検討されている | 設計の選択肢と選定理由が記載されている         |
| 技術的依存関係が明確         | 他モジュール・サービスへの影響が特定されている |
| UI/UX の決定がされている     | 該当する場合、画面設計・フローが確定している   |
| 分割案（子Issue）がある      | Medium/Small サイズの子Issueに分割されている   |

## サイズ判定ロジック

### 1. ラベル優先

以下のラベルがある場合、そのサイズとして判定:

- `size:small` → Small
- `size:medium` → Medium
- `size:large` → Large

### 2. 本文解析（ラベルがない場合）

| サイズ | 判定条件                                                                                 |
| ------ | ---------------------------------------------------------------------------------------- |
| Small  | 「バグ」「bug」「typo」「軽微」を含む、または本文500文字未満かつAC2項目以下              |
| Large  | 「アーキテクチャ」「設計変更」「大規模」を含む、または子Issue参照（`#` + 数字）が3つ以上 |
| Medium | 上記以外                                                                                 |

## Ready 判定基準

- **Ready**: 該当サイズの必須チェック項目を全て満たしている
- **Not Ready**: 1つ以上の必須チェック項目を満たしていない

## 関連ラベル

| ラベル                | 説明                                    |
| --------------------- | --------------------------------------- |
| `ready-for-work`      | DoRチェックをパスし、作業開始可能       |
| `needs-clarification` | DoRチェックで不足項目があり、確認が必要 |
