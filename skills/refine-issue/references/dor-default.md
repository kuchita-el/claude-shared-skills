# Definition of Ready (DoR)

Issueが作業開始可能かどうかを判定するための基準です。

これはデフォルト定義であり、最小限の共通項目のみを含みます。プロジェクト固有の要件（ACの必須化、技術設計の事前確定など）がある場合は `{project}/.claude/dor/definition.md` で拡張してください。

## チェック項目

全サイズ共通の項目です。

| 項目                                   | 説明                                                                                                  |
| -------------------------------------- | ----------------------------------------------------------------------------------------------------- |
| 課題・目的が明確                       | 何を解決/実現したいか、なぜ必要かがわかる                                                            |
| ゴールがわかる                         | 何をもって完了とするかの判断基準がある                                                                |
| 受入条件が検証可能な形で定義されている | 受入条件（AC）が1件以上記述され、各ACが「検証可能（verifiable）」かつ「客観的（objective）」である   |
| スコープの輪郭がある                   | 大まかな対象範囲（IN/OUT）が示されている                                                              |
| ブロッカーがない                       | 作業開始を阻む未解決の依存・前提がない                                                                |

「受入条件が検証可能な形で定義されている」は、AC自体の存在に加え、各ACの検証可能性・客観性まで含めて評価する。判定基準は後述「受入条件の検証可能性・客観性 判定基準」節を参照する。

## 受入条件の検証可能性・客観性 判定基準

精査ではAC毎に以下の2軸でそれぞれ判定する。AC全体ではなく**1項目ずつ**評価し、不合格項目があれば改善提案の対象とする。

### 検証可能（verifiable）

以下のいずれかを満たす場合、検証可能と判定する。

- 観測可能な事象（UI表示・ログ・APIレスポンス・ファイル出力等）として表現されている
- 自動テスト（ユニット／統合／E2E）に置き換えられる入出力対が記述可能
- 合否を客観判定できる入力と期待出力のペアが具体例として示せる
- 時間・回数・容量等の定量閾値が含まれる
- 状態変化の前後がそれぞれ定義されている

逆に、以下のような表現は検証不可と判定する。

- 「動作が安定する」「自然に使える」「期待通りに動く」等、観測対象が特定できない
- 「考慮する」「対応する」「サポートする」等、完了の境界が示されない
- 「適切に処理する」「正しく動く」等、判定基準を別途定義しないと合否判定できない

### 客観的（objective）

以下を全て満たす場合、客観的と判定する。

- 主観的な形容詞（満足／快適／安定／適切／自然／使いやすい／違和感がない 等）を含まない
- 評価者によって判断が分かれない表現になっている
- 定量・定性の閾値や境界が明記されている、もしくは別途参照する基準が指定されている

逆に、以下のような表現は主観的と判定する。

- 「ユーザーが満足する」「快適に動作する」等、評価者依存の判断を含む
- 「過度に遅くない」「許容範囲内」等、境界が明示されない
- 「使いやすい」「直感的」等、観測指標と切り離された定性表現

## サイズ判定ロジック

### 1. ラベル優先

以下のラベルがある場合、そのサイズとして判定:

- `size:small` → Small
- `size:medium` → Medium
- `size:large` → Large

### 2. 本文解析（ラベルがない場合）

| サイズ | 判定条件                                                                                 |
| ------ | ---------------------------------------------------------------------------------------- |
| Small  | 「バグ」「bug」「typo」「軽微」を含む、または本文500文字未満かつAC2項目以下              |
| Large  | 「アーキテクチャ」「設計変更」「大規模」を含む、または子Issue参照（`#` + 数字）が3つ以上、または設計の根本構造（責務境界・契約・アーキテクチャ原則）に踏み込む論点を含む |
| Medium | 上記以外                                                                                 |

#### 子Issue参照のカウント対象

「子Issue」「分割先Issue」として参照されているもののみカウント対象とする。「前提Issue」「関連Issue」「依存先」への言及はカウントしない。

## Ready 判定基準

- **Ready**: チェック項目を全て満たしている
- **Not Ready**: 1つ以上のチェック項目を満たしていない
