# Definition of Ready (DoR)

Issueが作業開始可能かどうかを判定するための基準です。

これはデフォルト定義であり、最小限の共通項目のみを含みます。プロジェクト固有の要件（ACの必須化、技術設計の事前確定など）がある場合は `{project}/.claude/dor/definition.md` で拡張してください。

## チェック項目

全サイズ共通の項目です。

| 項目               | 説明                                                     |
| ------------------ | -------------------------------------------------------- |
| 課題・目的が明確   | 何を解決/実現したいか、なぜ必要かがわかる               |
| ゴールがわかる     | 何をもって完了とするかの判断基準がある                   |
| スコープの輪郭がある | 大まかな対象範囲（IN/OUT）が示されている               |
| ブロッカーがない   | 作業開始を阻む未解決の依存・前提がない                   |

## サイズ判定ロジック

### 1. ラベル優先

以下のラベルがある場合、そのサイズとして判定:

- `size:small` → Small
- `size:medium` → Medium
- `size:large` → Large

### 2. 本文解析（ラベルがない場合）

| サイズ | 判定条件                                                                                 |
| ------ | ---------------------------------------------------------------------------------------- |
| Small  | 「バグ」「bug」「typo」「軽微」を含む、または本文500文字未満かつAC2項目以下              |
| Large  | 「アーキテクチャ」「設計変更」「大規模」を含む、または子Issue参照（`#` + 数字）が3つ以上 |
| Medium | 上記以外                                                                                 |

## Ready 判定基準

- **Ready**: チェック項目を全て満たしている
- **Not Ready**: 1つ以上のチェック項目を満たしていない

## 関連ラベル

| ラベル                | 説明                                    |
| --------------------- | --------------------------------------- |
| `ready-for-work`      | DoRチェックをパスし、作業開始可能       |
| `needs-clarification` | DoRチェックで不足項目があり、確認が必要 |
