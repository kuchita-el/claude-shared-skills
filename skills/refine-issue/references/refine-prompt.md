# Issue精査プロンプト

あなたはIssue精査の専門家です。以下の手順でIssueを精査してください。

## 精査の目標

「このIssueを渡された開発者が、追加の質問なしに作業を始められるか？」を判断すること。

## サイズ判定

DoR定義の「サイズ判定ロジック」セクションに従ってサイズを判定する。

1. **ラベル優先**: `size:small`, `size:medium`, `size:large` ラベルがあればそのサイズ
2. **Large判定**: 本文に「アーキテクチャ」「設計変更」「大規模」を含む、または子Issue参照（`#` + 数字）が3つ以上
   - 子Issue参照のカウント: 「子Issue」「分割先Issue」として参照されているもののみ。「前提Issue」「関連Issue」「依存先」への言及はカウントしない
3. **Small判定**: 本文に「バグ」「bug」「typo」「軽微」を含む、または本文500文字未満かつAC2項目以下
4. **上記いずれにも該当しない場合**: Medium

## DoRチェック

判定したサイズに応じたDoRチェック項目を全て評価する。

## 再精査時の注意

精査コメント（「精査実施: Claude Code」を含む）とそれ以降のユーザー回答が存在する場合は再精査である。

- ユーザー回答で解消済みの確認事項は繰り返さない
- 回答内容を踏まえて新たに判明した確認事項のみ提示する
- 回答により方針が変わった場合、Issue本文との乖離も指摘する
