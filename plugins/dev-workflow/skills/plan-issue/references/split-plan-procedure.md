# 複数 PR 分割対応の手順（plan-issue ハイブリッドモード）

plan-issue SKILL.md のステップ1（モード判定）から参照される手順。通常モードの判定後、分割計画ファイルの有無で初回/N回目を判定し、処理を分岐する。

## 判定基準

**分割計画マスターファイル**（`docs/plans/issue-{番号}-plan.md`）の有無で判定する:

| 分割計画マスターファイルの状態 | 判定 |
|---|---|
| `issue-{番号}-plan.md` が存在しない | **初回モード** |
| `issue-{番号}-plan.md` が存在する | **N回目モード** |

### 既存ステップ6（連番サフィックス衝突回避）との区別

ステップ6は `issue-{番号}.md`（詳細プランファイル）の有無を判定して連番サフィックスを付与する（人間レビュー却下後の再生成リカバリ用）。本手順は `issue-{番号}-plan.md`（分割計画マスター）の有無を判定する。**判定対象ファイルが異なるため両者は衝突しない**:

- `issue-{番号}-plan.md` の有無 → 本手順（初回/N回目モード）の判定に使用
- `issue-{番号}.md` の有無 → ステップ6（連番リカバリ）の判定に使用

## 初回モード（分割計画マスターが存在しない場合）

N プラン分割計画と1本目の詳細プランを生成する:

1. AC↔PR マトリクスを含む **N プラン分割計画**を `docs/plans/issue-{番号}-plan.md` に生成する
   - 全 AC をどの PR が担当するかのマトリクス
   - PR 間の前後依存
   - 各 PR のスコープ概要（担当 AC・主な変更対象）
2. **1本目の詳細プラン**をステップ5〜6 の通常フローで生成し `docs/plans/issue-{番号}.md` に保存する
   - 出力フォーマットの「分割の文脈」セクションを含める（`${CLAUDE_SKILL_DIR}/references/plan-output-format.md` 参照）

**遷移例**: `docs/plans/issue-268-plan.md` が存在しない状態で `plan-issue 268` を実行
→ `issue-268-plan.md`（3PR 分割計画マスター）と `issue-268.md`（PR-A 詳細プラン、分割の文脈セクション入り）を生成

## N回目モード（分割計画マスターが存在する場合）

引数で指定された k 本目のプランを詳細化する:

1. `docs/plans/issue-{番号}-plan.md` を Read して分割計画を確認する
2. k 本目の担当 AC・スコープ・前後依存を分割計画から読み取る
3. k 本目の詳細プランをステップ5〜6 の通常フローで生成し `docs/plans/issue-{番号}-k.md` に保存する
   - 「分割の文脈」セクションに k 本目の情報を記入する
   - ステップ6 の衝突回避機構（`issue-{番号}-k.md` の既存確認→連番サフィックス）は変わらず適用される

**遷移例**: `docs/plans/issue-268-plan.md` が存在する状態で k=2 を指定して `plan-issue 268` を実行
→ `issue-268-2.md`（PR-B 詳細プラン）を生成

## k（何本目か）の受け取り方

N回目モード起動時、補足指示として k を受け取る（例: `plan-issue 268 "k=2"` や `plan-issue 268 2本目`）。k の指定がない場合は、分割計画マスターを参照して未詳細化の PR をユーザーに提示し、どの PR を詳細化するか確認する。
