# PR 本文テンプレート（Phase 4 セルフレビュー結果のサマリ）

implementation の Phase 4 で `superpowers:finishing-a-development-branch` 等の委譲出力に上乗せする、セルフレビュー結果サマリのテンプレート。本体 SKILL.md の Phase 4 節からポインタ参照される。

## テンプレート

PR 本文に以下の順序でセクションを記載する。`## 対応 Issue AC` をセルフレビューサマリより先に挿入する（生成方法は「対応 Issue AC セクション」の3分岐を参照。分岐3の場合はセクションごと省略し、`## セルフレビュー実施済み` から開始する）。

```markdown
## 対応 Issue AC

- AC4
- AC5

## セルフレビュー実施済み

レビュー: {N}周実施、ブロッカー{M}件を修正済み

### 修正した指摘
| 重大度 | 指摘 | 対応内容 | コミット |
|---|---|---|---|
| ブロッカー | ... | ... | ... |

### 深刻度調整（誤検知格下げ）
| 調整 | 指摘 | 格下げ根拠（周辺コードの file:line） |
|---|---|---|
| ブロッカー → 改善提案 | ... | src/middleware/auth.ts:45（既にバリデーション済み） |
```

## 深刻度調整セクションの位置づけ

Phase 2 の深刻度調整で `ブロッカー → 改善提案` へ格下げした指摘は、純粋な改善提案と区別するため「深刻度調整（誤検知格下げ）」セクションに上記テーブル形式で要約する（各行＝1格下げ指摘、根拠は周辺コードの `file:line`）。

このテーブルは、Phase 2 の調整出力（`${CLAUDE_SKILL_DIR}/references/review-severity-adjustment.md` のブロック形式＝元の指摘を保持するワーキング出力）を **人間レビュアー向けに要約したもの**であり、両者は役割が異なる:

- **`review-severity-adjustment.md` のブロック形式**: 作業出力（元の指摘を保持）
- **本テンプレートの「深刻度調整（誤検知格下げ）」セクション**: PR 本文向け要約

格下げが無い場合は当該セクションを省略する。

## 対応 Issue AC セクション

PR 本文に `## 対応 Issue AC` セクションを挿入する。生成元は以下の3分岐で決定する:

1. **起点プランに「分割の文脈」がある場合**: 「分割の文脈 > 対応 Issue AC のリスト」の内容を各ACを1行の箇条書きとして転記する
2. **「分割の文脈」がなく Issue 起点の場合**: `gh issue view` で取得した Issue 本文のAC全項目を列挙する（単一PRプランでは全ACがそのPRの責務）。デフォルトでは Issue 本文の `## 受け入れ条件` セクション下の箇条書き項目をACとして列挙する。プロジェクト固有のAC形式がある場合は `.claude/dor/definition.md` 等を参照し上書きする
3. **「分割の文脈」も Issue 情報もない場合（会話コンテキスト起点等）**: セクション自体を省略する（空セクション・プレースホルダを残さない）

**入出力例（分割の文脈ありの場合）**:

プランの「対応 Issue AC のリスト」: `AC4, AC5`

```markdown
## 対応 Issue AC

- AC4
- AC5
```

## 改善提案セクション

その他の改善提案（深刻度調整による格下げではない、純粋な改善提案）は「改善提案（未対応）」セクションに記載し、人間のレビュアーに判断を委ねる。
