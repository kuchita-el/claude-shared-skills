# 委譲テーブル

「解く」の委譲先を、調査の種類（`kind`）ごとに引き当てるためのデータである。**具体的なツール名が現れる唯一のファイル**であり、委譲先の差し替え単位になる（ADR-20260716 決定2「inquiry 本体は不変に保つ」）。力不足が判明した場合は当該行の第2列・第3列のみを差し替え、`SKILL.md` 本体には手を入れない。

## 委譲インターフェース

委譲の入力は「**問い ＋ 必要な調査の種類（`kind`）**」の対とする（ADR-20260716 決定2・AC9）。加えて、根の問いの原文を verbatim で添える（AC6。改変・要約・言い換えをしない）。

`kind` は台帳のエントリがステップ1（分解）の時点で既に保持している値であり、委譲の時点で新たに判定するものではない。**どの問いがどの `kind` に落ちるかの判定の手がかりと `kind` の値域の定義は `SKILL.md` 本体の「調査の種類（`kind`）の値域」節が単一出典**であり、本ファイルは判定を行わない。本ファイルは「本体が決めた `kind` を委譲先へ引き当てる表」に純化している。

## テーブル

第1列＝調査の種類、第2列＝委譲先、第3列＝起動手段の3列。**第1列は `SKILL.md` 本体の「調査の種類（`kind`）の値域」節の7語を結合キーとして並べたものであり、値域を定義するものではない**（定義は本体。AC9 は本体の値域7語と本テーブル第1列の一致を要求する）。

| 調査の種類（`kind`） | 委譲先 | 起動手段 |
|---|---|---|
| コードベース内の事実調査 | `Explore`（組み込みエージェント） | `Agent` ツールで起動 |
| Web・文献調査 | `deep-research` | `Skill` ツールで起動 |
| 依存・互換性調査 | `dependency-check` | `Skill` ツールで起動 |
| ドメイン理解・モデリング | `event-storming`（戦略的洗い出し）／`domain-modeling`（型・状態・ワークフローの形式化） | `Skill` ツールで起動 |
| 履歴・経緯調査 | `general-purpose`（汎用エージェント。専用手段が空白のため） | `Agent` ツールで起動 |
| 実験・PoC | `general-purpose`（汎用エージェント。専用手段が空白のため） | `Agent` ツールで起動 |
| 汎用 | `general-purpose`（汎用エージェント。**縮退先**でもある） | `Agent` ツールで起動 |

- 空白領域（履歴・経緯調査、実験・PoC）と、どの種別にも当てはまらない問いを受ける「汎用」は、いずれも汎用エージェントを委譲先とする（ADR-20260716 決定2）。
- **縮退先**: 「汎用」の行が縮退先である。ある行の委譲先が実行環境に不在のとき、この行の委譲先へ落ちる。**縮退の規則そのもの（いつ縮退するか・調査を継続し異常終了しないこと）は `SKILL.md` 本体の「委譲」節が単一出典**であり、本ファイルへは複写しない（本体が規則、本テーブルがデータという分担）。ADR-20260606-3 のソフト依存方針と整合する。
- **起動手段列**: 委譲先が組み込みエージェント（`Agent` ツール）か スキル（`Skill` ツール）かで起動ツールが異なるため、起動手段を列のデータとして持つ。これにより `SKILL.md` 本体は `Agent` / `Skill` の分岐を持たずに済む（AC9 の不変条件）。

## 入出力の対応例

- **入力**: 問い＝「この関数の呼び出し元はどこか」、`kind`＝`コードベース内の事実調査`（ステップ1 で台帳へ付与済みの値を読む）、根の問いの原文（verbatim で添える）。
- **引き当て**: 第1列 `コードベース内の事実調査` の行を引く → 委譲先 `Explore`、起動手段「`Agent` ツールで起動」。
- **出力**: 当該行の起動手段に従って `Agent` ツールで `Explore` を起動し、問い・根の問いの原文を渡す。

## 不変条件

- 本テーブルは委譲先の差し替え単位である。力不足が判明した場合は当該行の第2列・第3列のみを差し替え、`SKILL.md` 本体を不変に保つ（ADR-20260716 決定2）。
- 具体的なツール名（委譲先・起動ツール）は本ファイルにのみ現れる。`SKILL.md` 本体の分岐条件には現れない（AC9）。
